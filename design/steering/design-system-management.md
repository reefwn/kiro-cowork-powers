# Design System Management

Manage design tokens, component libraries, and pattern documentation.

## When This Applies

Loaded when discussing design tokens, component consistency, naming conventions, or pattern documentation.

## Key Areas

- **Tokens**: Colors, typography, spacing, borders, shadows, motion
- **Components**: Variants, states, sizes, behavior, accessibility
- **Patterns**: Forms, navigation, data display, feedback

## Principles

1. Consistency over creativity
2. Flexibility within constraints
3. Document everything
4. Version and migrate breaking changes
