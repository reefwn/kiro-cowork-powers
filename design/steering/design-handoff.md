# Design Handoff

Create comprehensive developer handoff documentation from designs.

## When This Applies

Loaded when preparing designs for engineering implementation, creating spec sheets, or documenting component behavior for developers.

## What to Specify

- **Visual**: Exact measurements, token references, responsive breakpoints
- **Interaction**: Click/tap, hover, transitions, gestures
- **Content**: Character limits, truncation, empty/loading/error states
- **Edge cases**: Min/max content, i18n, slow connections, missing data
- **Accessibility**: Focus order, ARIA labels, keyboard interactions

## Principles

1. Don't assume — specify everything
2. Use tokens, not raw values
3. Show all states
4. Describe the why for context
