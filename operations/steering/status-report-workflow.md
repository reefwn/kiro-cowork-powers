# Status Report Workflow

Generate a polished status report for leadership or stakeholders.

## When to Use
When the user needs a weekly, monthly, or quarterly status report with KPIs, risks, and action items.

## Output

```markdown
## Status Report: [Project/Team] — [Period]
**Author:** [Name] | **Date:** [Date]

### Executive Summary
[3-4 sentence overview — what's on track, what needs attention, key wins]

### Overall Status: 🟢 On Track / 🟡 At Risk / 🔴 Off Track

### Key Metrics
| Metric | Target | Actual | Trend | Status |
|--------|--------|--------|-------|--------|

### Accomplishments This Period
- [Win 1]
- [Win 2]

### In Progress
| Item | Owner | Status | ETA | Notes |
|------|-------|--------|-----|-------|

### Risks and Issues
| Risk/Issue | Impact | Mitigation | Owner |
|------------|--------|------------|-------|

### Decisions Needed
| Decision | Context | Deadline | Recommended Action |
|----------|---------|----------|-------------------|

### Next Period Priorities
1. [Priority 1]
2. [Priority 2]
3. [Priority 3]
```

See the **risk-assessment** steering for risk matrix frameworks and severity definitions.

## If Connectors Available

- **Project tracker**: Pull project status, completed items, upcoming milestones; identify at-risk and overdue items
- **Chat**: Scan team discussions for decisions and blockers; offer to post report to a channel
- **Calendar**: Reference key meetings and decisions from the reporting period

## Tips

- Lead with the headline — busy leaders read the first 3 lines
- Be honest about risks — surfacing issues early builds trust
- Make decisions easy — provide context and a recommendation for each
